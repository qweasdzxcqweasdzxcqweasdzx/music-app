## 🔧 РУЧНОЙ ЗАПУСК


**Должно вернуть:**

### 2️⃣ Запуск CORS Proxy (порт 8081)

**Откройте НОВОЕ окно терминала:**

```bash
cd /home/c1ten12/music-app/backend
source venv/bin/activate
python cors_proxy_8081.py
```

**Проверка:**
```bash
curl http://localhost:8081/api/censorship/test
```

**Должно вернуть:**

### 3️⃣ Запуск Cloudflare Tunnel

**Откройте ТРЕТЬЕ окно терминала:**


**Дождитесь URL вида:**
```

```

**Скопируйте этот URL!**

## 🤖 АВТОМАТИЧЕСКИЙ ЗАПУСК

### Скрипт start-with-url-update.sh

**Расположение:** `/home/c1ten12/music-app/start-with-url-update.sh`

**Использование:**
```bash
/home/c1ten12/music-app/start-with-url-update.sh
```

**Что делает:**
1. Проверяет и запускает сервер если не работает
2. Проверяет и запускает CORS proxy если не работает
3. Запускает Cloudflare Tunnel
4. Получает новый HTTPS URL
5. Обновляет `frontend/src/api/musicApi.js`
6. Обновляет `frontend/src/pages/Search.jsx`
7. Собирает фронтенд (`npm run build`)
8. Коммитит и пушит на GitHub

**Выход:**
```
✅ Cloudflare запущен
🌐 URL: https://xxxx-xxxx-xxxx.trycloudflare.com
✅ Фронтенд обновлён
✅ ВСЁ РАБОТАЕТ!
```

## 📡 API ENDPOINTS

### Основные

| Endpoint | Метод | Описание |
|----------|-------|----------|
| `/health` | GET | Проверка здоровья сервера |
| `/api/censorship/test` | GET | Тест Anti-Censorship системы |
| `/api/censorship/check` | GET | Проверка трека на цензуру |
| `/api/censorship/search-uncensored` | GET | Поиск с explicit приоритетом |
| `/api/censorship/find-original` | POST | Поиск оригинальной версии |

### Примеры

**Проверка здоровья:**
```bash
curl http://localhost:8000/health
```

**Поиск треков:**
```bash
curl "http://localhost:8000/api/censorship/search-uncensored?q=eminem&limit=5"
```

**Проверка на цензуру:**
```bash
curl "http://localhost:8000/api/censorship/check?track_id=123"
```

## 🔧 УСТРАНЕНИЕ ПРОБЛЕМ

### Проблема 1: Cloudflare ошибка 530

**Симптом:** `Error 530 - Origin not registered`

**Решение:**
```bash
# Перезапустите Cloudflare
pkill -f cloudflared
/home/c1ten12/music-app/start-with-url-update.sh
```

---

### Проблема 2: CORS ошибка в браузере

**Симптом:** `Access to fetch blocked by CORS policy`

**Решение:**
```bash
# Проверьте что CORS proxy работает
curl http://localhost:8081/api/censorship/test

# Если не работает - перезапустите
pkill -f cors_proxy
cd /home/c1ten12/music-app/backend
source venv/bin/activate
python cors_proxy_8081.py &
```

---

### Проблема 3: Сервер не запускается

**Симптом:** `Address already in use`

**Решение:**
```bash
# Убейте старые процессы
pkill -f uvicorn
pkill -f cors_proxy
pkill -f cloudflared

# Запустите заново
/home/c1ten12/music-app/start-with-url-update.sh
```

---

### Проблема 4: GitHub Pages не обновляется

**Симптом:** Старая версия на GitHub Pages

**Решение:**
```bash
# Подождите 1-2 минуты
# GitHub Pages требует времени на обновление

# Или проверьте что пуш прошёл
cd /home/c1ten12/music-app
git log --oneline -3
```

---

### Проблема 5: Cloudflare постоянно отключается

**Симптом:** Cloudflare требует частого перезапуска

**Решение:**
- Это нормально для бесплатных Cloudflare Quick Tunnels
- Используйте скрипт `start-with-url-update.sh` для быстрого перезапуска
- Для продакшена рассмотрите платный Cloudflare Tunnel
